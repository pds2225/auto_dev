# 리드 → 구매 전환율 극대화 프론트엔드 설계 문서

> **버전:** v1.0  
> **대상:** Auto Dev 대시보드 (Streamlit 기반)  
> **목표:** 방문자 → 활성 사용자 → 유료 전환 퍼널 최적화  
> **반영 여부:** ❌ 설계 단계 (미반영)

---

## 1. Executive Summary

현재 Auto Dev 대시보드는 **"개발자용 내부 도구"** 중심 UI로 구성되어 있음.  
외부 사용자(잠재 고객)가 처음 접했을 때 **가치 인지 → 신뢰 형성 → 전환**까지의 여정이 명확하지 않음.

### 핵심 지표 (KPI)
| 단계 | 현재 추정 | 목표 |
|------|----------|------|
| 방문자 → 랜딩 페이지 체류 60초 | 35% | 70% |
| 랜딩 → 데모 실행 | 5% | 20% |
| 데모 → 회원가입/로그인 | 10% | 35% |
| 가입 → 유료 플랜 전환 | 1% | 8% |

---

## 2. 사용자 세그먼트 & 페르소나

### 2.1 Primary — "빠른 결과 원하는 개발자 리드"
- **특성:** 사이드 프로젝트 진행 중, AI 자동화에 관심 있지만 시간 없음
- **통증:** TASK.md 관리, Claude Code 반복 명령어, 테스트/디버그 루틴
- **목표:** 5분 내로 "이 도구가 내 프로젝트를 대신 돌려준다"는 확신 얻기

### 2.2 Secondary — "팀 도구를 찾는 테크 리드"
- **특성:** 팀 생산성 도구 구매 결정권 보유
- **통증:** 팀원들이 반복적 하드닝/테스트에 시간 소모
- **목표:** 팀 전체 도입 가능성과 ROI를 3분 내로 확인

### 2.3 Tertiary — "무료로 써보려는 호기심 리드"
- **특성:** GitHub 레포 발견 → Streamlit 링크 클릭 → 가벼운 시도
- **통증:** 설치 없이 바로 체험하고 싶음
- **목표:** 설치/가입 없이 데모 시연 영상 또는 샌드박스 체험

## 3. 정보 아키텍처 (IA) 재설계

### 3.1 현재 구조 (As-Is)
```
[로그인 없음] → 404 / Cloudflare Error
[대시보드] → 사이드바(경로입력) + 메인(태스크+로그+도구)
└── 딱히 "가입 유도" 흐름 없음
```

### 3.2 제안 구조 (To-Be)
```
/                    → 랜딩 페이지 (가치 제안 + CTA)
/demo                → 인터랙티브 데모 (로그인 없이 체험)
/dashboard           → 대시보드 (로그인 후)
/pricing             → 가격 정책 (전환 최적화)
/docs                → 문서/온보딩
```

## 4. 랜딩 페이지 설계 (/)

### 4.1 히어로 섹션 (Above the Fold)
```
┌─────────────────────────────────────────┐
│  🤖 Auto Dev                            │
│  "AI가 당신의 개발 반복 작업을 끝내줍니다" │
│                                         │
│  [▶ 30초 데모 보기]    [무료로 시작하기]  │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │  [실제 대시보드 GIF/영상]        │    │
│  │  태스크 자동 생성 → 루프 실행     │    │
│  │  → 테스트 통과 → 완료             │    │
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

**핵심 원칙:**
- **H1:** 기능 명시가 아닌 "결과" 명시 ("AI가 개발 반복 작업을 끝내준다")
- **영상:** 실제 루프가 도는 화면 5초짜리 GIF, 총 8초
- **CTA 2개:** 관찰형(영상) vs 행동형(시작) — 선택지 제공 = 진입 장벽 완화

### 4.2 사회적 증명 (Social Proof)
```
┌─────────────────────────────────────────┐
│  "이미 00개의 프로젝트가 Auto Dev로    │
│   자동화되어 있습니다"                 │
│                                         │
│  [GitHub stars: ★ 12] [커밋 수 뱃지]   │
│  [Discord 참여자 수]                   │
└─────────────────────────────────────────┘
```

### 4.3 가치 제안 3단계
```
┌──────────┬──────────┬──────────┐
│  1. 입력  │  2. 실행  │  3. 완료  │
│  자연어   │  AI 하드닝 │  테스트   │
│  태스크   │  + 디버그  │  자동 통과│
│  설명     │           │           │
└──────────┴──────────┴──────────┘
```

## 5. 인터랙티브 데모 페이지 (/demo)

**핵심 전략:** 설치 없이 "체험" → 가치 인지 → 가입 유도

### 5.1 샌드박스 모드
- **사이드바:** 기술스택 선택 (Streamlit / FastAPI / Python)
- **입력:** "대시보드에 다크모드 토글 버튼 추가" 같은 예시 문구 제공
- **출력:** AI가 생성한 태스크 리스트 + 예상 소요 시간 (시뮬레이션)
- **CTA:** "이 결과를 실제로 실행하려면? → 무료 시작"

### 5.2 데모 → 전환 터널
```
[데모 결과 확인] → "이 태스크를 지금 실행해보세요" → [이메일 입력] → [가입 완료]
                    ↑
              (가입 없이 결과 3개까지만 보여줌)
```

## 6. 대시보드 내부 전환 지점 (Dashboard)

현재 대시보드에는 전환(구매) 관련 UI가 전혀 없음.  
**"프리미엄 기능 잠금 + 업그레이드 유도"** 패턴을 도입.

### 6.1 프리미엄 기능 차단 (Freemium Gate)
| 기능 | Free | Pro |
|------|------|-----|
| 태스크 자동 생성 (AI 분해) | 월 5회 | 무제한 |
| 루프 자동 예약 실행 | ❌ | ✅ |
| 다중 프로젝트 큐 관리 | 1개 | 무제한 |
| Claude Code 하드닝 | ❌ | ✅ |
| Telegram 알림 | ❌ | ✅ |

### 6.2 게이트 UI 패턴
```python
# 제안 컴포넌트 (미반영 — 설계만)
if task_limit_reached():
    st.info("🚀 Pro로 업그레이드하면 AI 태스크 생성을 무제한으로 사용할 수 있습니다")
    st.button("💎 Pro 시작하기 — $9/월", type="primary")
```

### 6.3 진행 상황 게이지 (진행률 = 몰입도 = 전환율)
```
┌────────────────────────────────────┐
│  🎯 프로젝트 진행률 23%            │
│  [████████░░░░░░░░░░░░]            │
│  "Pro로 업그레이드하면 자동으로      │
│   나머지 77%도 완료됩니다"          │

```

## 8. 온보딩 퍼널 (Onboarding Funnel)

### 8.1 단계별 리텐션 설계
```
Day 0: 가입 완료 → 축하 메시지 + "첫 프로젝트 경로를 입력해보세요"
Day 1: 이메일 — "첫 태스크를 생성해보셨나요?" + 태스크 생성 가이드
Day 3: 이메일 — "루프를 실행하면 AI가 자동으로 테스트까지 해줍니다" + 동영상
Day 7: 이메일 — "Pro 기능 미리보기" + 할인 쿠폰 (20% off 첫 달)
Day 14: 이메일 — "지난주 00개의 태스크가 자동 완료되었습니다" + 업그레이드 CTA
```

### 8.2 대시보드 내 온보딩 체크리스트
```
┌────────────────────────────────────────┐
│  👋 시작하기 (3/4 완료)                │
│  ✅ 프로젝트 경로 설정                 │
│  ✅ 첫 태스크 생성                     │
│  ⬜ 루프 실행해보기  ← 현재 단계        │
│  ⬜ 예약 설정하기                      │
│                                        │
│  [다음 단계: 루프 실행하기]            │
└────────────────────────────────────────┘
```

## 9. 신뢰 형성 (Trust Signals)

### 9.1 대시보드 내 배치
- **실시간 상태 뱃지:** "● 3개 프로젝트가 지금 자동 실행 중"
- **로그 투명성:** 실패/성공 모두 표시 (숨기지 않음 = 신뢰)
- **GitHub 연동 상태:** "origin: github.com/pds2225/auto_dev" → 클릭하면 stars/commits 노출

## 10. A/B 테스트 로드맵

| 테스트 | 가설 | 지표 | 우선순위 |
|--------|------|------|----------|
| H1 문구 | "AI가 개발 반복 작업을 끝내줍니다" vs "자동화된 개발 워크플로우" | 클릭률 | P0 |
| CTA 색상 | 초록 vs 파랑 vs 보라 | 전환율 | P0 |
| 데모 위치 | 히어로 내 vs 아래 | 체류 시간 | P1 |
| 프리미엄 게이트 위치 | 5회 차단 vs 10회 차단 | 유료 전환 | P1 |
| 가격 표시 | 월간 기본 vs 연간 기본 | ARPU | P2 |

## 11. 구현 우선순위

### Phase 1 — 랜딩 & 데모 (전환율 +200% 예상)
1. 루트 경로(`/`)를 랜딩 페이지로 변경
2. `/demo` 샌드박스 모드 구현
3. `/pricing` 3티어 페이지 추가
4. 이메일 수집 게이트 추가

### Phase 2 — 프리미엄 게이트 (MVP 수익화)
5. AI 태스크 생성 횟수 카운팅
6. 5회 도달 시 업그레이드 모달
7. Pro 기능 잠금 (예약 실행, Telegram, 다중 큐)

### Phase 3 — 온보딩 & 리텐션 (LTV 향상)
8. 온보딩 체크리스트 UI
9. 이메일 자동 시퀀스 연동
10. 사용량 기반 트리거 알림

## 12. Streamlit 한계 & 대안

### 12.1 Streamlit으로 가능한 것
- 랜딩 페이지 (st.markdown + st.button + custom CSS)
- 프리미엄 게이트 (st.session_state + 조건부 렌더링)
- 가격 페이지 (st.columns + st.metric)

### 12.2 Streamlit의 한계
- **SEO:** SPA라서 검색엔진 최적화 어려움
- **URL 라우팅:** `/pricing` 같은 경로 처리 복잡 (multipage로 해결 가능)
- **디자인 자유도:** 상업적 랜딩 페이지 수준의 디자인 한계

### 12.3 대안 제안
만약 진지하게 상용화한다면:
- **랜딩/마케팅:** Next.js / Astro (정적 사이트) — `autodev.app`
- **대시보드:** Streamlit 유지 — `app.autodev.app`
- **API:** FastAPI — `api.autodev.app`
