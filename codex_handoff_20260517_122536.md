## 1. 기본 원칙

- **최소 변경 원칙**: 목표 달성에 필요한 최소한의 변경만 수행한다.
- **기존 기능 보호**: 작동 중인 기능을 절대 깨뜨리지 않는다.
- **안전 우선**: 검증되지 않은 변경은 실제 파일에 적용하지 않는다.
- **투명한 로그**: 수행한 작업, 실패 이유, 다음 행동을 로그에 명시한다.
- **Secret 보호**: API Key, Token, 비밀번호 값은 절대 출력하지 않는다.

## 공통 작업 지침 (CLAUDE.md)
# Claude Code 공통 작업 지침

- 사용자는 Windows PowerShell에서 실행한다.
- Python 실행은 `python ...` 형식으로 안내한다.
- `streamlit`, `pytest`, `pip`처럼 PATH 영향이 있으면 `python -m ...` 형식을 우선한다.
- 작업 시작 전에 현재 폴더와 실제 repo root를 확인한다.
- repo 안에 `AGENTS.md`, `RULES.md`, `CLAUDE.md`, `CODEX.md`가 있으면 먼저 읽는다.
- 공통 지침과 repo-local 규칙이 충돌하면 repo-local 규칙을 우선한다.
- 사용자가 "수정하지 마", "계획만", "원인만"이라고 하면 파일을 수정하지 않는다.
- 전체 파일을 무작정 읽지 말고 관련 파일만 최소로 확인한다.
- 필요한 파일만 수정하고 기존 사용자 변경사항은 되돌리지 않는다.
- 요구하지 않은 리팩토링이나 기능 추가는 하지 않는다.
- 답변 첫 줄에는 상태를 표시한다: 정상 실행 확인됨 / 수정만 완료 / 미검증 / 실행 막힘 / 수정 없음.
- 파일을 수정한 경우 수정 파일, 이유, 실행/검증 명령어, 검증 결과를 함께 보고한다.
- 커밋 전에 Secret, API Key, Token, 비밀번호가 코드나 로그에 없는지 확인한다.
